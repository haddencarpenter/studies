The Privacy Finding
Environment leak

A show env keys prompt returned both NEXT_PUBLIC_SHUMI_API_ENDPOINT and NEXT_PUBLIC_SHUMI_SUGGESTIONS_ENDPOINT with their full URL values from .env.example. These are NEXT_PUBLIC_ prefixed, so they ship in the browser bundle anyway — but the operator's point stands: the assistant had no gate for "am I being recorded right now?"

Behavioral fix

The fix is a scoped memory entry, not code. The assistant now distinguishes recording mode from normal mode and suppresses config values when the operator signals they are recording. Normal sessions show everything. The memory was over-scoped on the first try (all sessions), then corrected (recording only).

Why This Is Published

Three corrections in five minutes is a normal Claude Code interaction, not a failure. The first correction caught the leak. The second scoped the fix. The third redirected the session toward the actual task.

Each correction required the assistant to re-read its own memory, update it, and re-orient. The operator never explained the full context upfront. This is how real context engineering works: incremental steering, not upfront specification.

Most AI product demos show curated, successful interactions. This study publishes the rough edges: an assistant that leaks values, a context recovery that takes multiple hops, and a user correcting the AI three times before it understands the request. The session transcript is the proof.
